## 可用组件（汉字 → 含义）

| 字 | 组件 | 可用属性及顺序说明 |
|---|---|---|
| 列 / 排 | 纵向/横向容器 | 无属性 |
| 卡 | Card | 无属性 |
| 域 | Form | 触（onSubmit action） |
| 文 | Text | 式（标题/主标题/正文/价格/节标题/说明） 值前缀 |

| 选 | Select | 值前缀 选项前缀 必 禁 |

## 特殊前缀语法

**只有任务明确提及时才用以下前缀**，静态内容不需要任何前缀：
- `値xxx` → 绑定状态变量 xxx（如 `値email` = bind="email"），**仅当任务描述了需要绑定的数据时才用**
- `选项xxx` → 选项列表变量名 xxx（仅 Select 用：`选项categories`）
- `触xxx` → 触发 action xxx（如 `触submit`），仅用于 Form 组件的 onSubmit
- `空xxx` → placeholder xxx
- `必` → required=true
- `禁` → disabled=true

**常见的错误：**
- 静态文本不需要 値 前缀！如 `文 标题 用户名`（正确），不要写 `文 标题 値username`（错误）
- Badge 的静态内容直接写文字：`标 成功 5.0`（正确），不要写 `标 成功 値rating`（错误）

## Text variant 写法
Text 的 variant 是裸字符，**不要加 式 作前缀**：
- ✅ 正确：`文 标题 确认操作`
- ❌ 错误：`文 式 标题 确认操作`

## Input 类型写法
Input 的 type 是裸字符，**不要加 类 作前缀**。只有需要限制输入格式时才写 type：
- ✅ 正确：`入 邮箱 必 空请输入邮箱 値email`
- ❌ 错误：`入 类 邮箱 必 空请输入邮箱 値email`

## 缩进规则
- 用全角空格（　）缩进，每层 1 个全角空格
- 子节点放在父节点下一行，缩进更深
- 不需要 []、{}、","

## 输出要求
只输出行式 wire 代码，不要 markdown 代码块，不要解释，不要中文注释。

## 绑定值（値）命名规则
- 値xxx 中的变量名使用任务描述中明确指定的名字，不要自己翻译
- 任务说"绑定 title 状态" → 使用 値title（对号入座，不翻译成"标题"）
- 任务说"绑定 email 状态" → 使用 値email
- 任务说"绑定 username 状态" → 使用 値username
- 严格使用 prompt 里的英文原名，不要根据名词含义翻译成中文
